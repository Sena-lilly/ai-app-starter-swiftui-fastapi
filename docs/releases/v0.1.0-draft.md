## Summary

`ai-app-starter-swiftui-fastapi` v0.1.0 is the first public baseline for a production-minded, local-first starter kit for indie developers building iOS apps with SwiftUI, FastAPI, PostgreSQL, JWT authentication, Docker Compose, and Codex-safe workflows.

This release is intended as a practical reference implementation and workflow template. It is not a production-ready application.

## Highlights

- SwiftUI + FastAPI starter kit for local-first iOS app development.
- Minimal backend auth flow with JWT access tokens.
- SwiftUI auth flow with Keychain token storage.
- Optional local-only Docker Compose/PostgreSQL setup with Alembic migrations.
- Codex-safe workflow docs, templates, review prompts, and handoff guidance.
- CI foundation for backend tests and iOS simulator build.
- Docs/script CI checks for shell syntax and local Markdown links.
- Expanded backend auth/security/config/session test coverage.
- Release-readiness, secret-audit, and example walkthrough documentation.
- GitHub Issue/PR templates for public maintenance.
- Lightweight local-only iOS XCTest foundation.

## Safety Notes

- This starter is production-minded, not production-ready.
- Do not use placeholder credentials in production.
- Do not connect to production or external databases during starter verification.
- Do not use real user credentials in local examples.
- Do not run `docker compose down -v` unless you explicitly intend to delete the local PostgreSQL volume.
- GitHub releases, tags, deploys, pushes, and App Store/TestFlight operations require explicit human confirmation.

## Release Verification Checklist

The v0.1.0 release was expected to be published only after confirming:

- Backend tests pass.
- iOS simulator build passes.
- iOS XCTest passes, or `build-for-testing` passes and any local Simulator limitation is documented.
- Secret audit passes.
- Shell script syntax check passes.
- Markdown link check passes.
- Docker/PostgreSQL local verification passes when Docker is available.
- GitHub Actions are green for the release commit.
- No `.env`, local DB files, DerivedData, caches, logs, production URLs, or real secrets are included.

Local verification commands:

```bash
scripts/preflight-local.sh
```

Optional local runtime verification:

```bash
scripts/preflight-local.sh --with-ios-tests
scripts/preflight-local.sh --with-docker
```

Manual equivalent:

```bash
scripts/secret-audit.sh
bash -n scripts/*.sh
python3 scripts/check-markdown-links.py
scripts/local-verify-backend.sh
scripts/local-verify-ios.sh
scripts/local-verify-docker.sh
```

## Upgrade / Migration Notes

- This is the first public release, so there is no user-facing upgrade path yet.
- Local Docker/PostgreSQL users should run `docker compose exec backend alembic upgrade head`.
- Do not delete local Docker volumes unless you intentionally want to remove local test data.

## Historical Decision Note: v0.1.0 vs v0.1.0-pre

Use `v0.1.0` when backend tests, iOS build, local-only iOS tests or `build-for-testing`, Docker/PostgreSQL verification, secret audit, generated artifact audit, and GitHub Actions are all reviewed.

Use `v0.1.0-pre` if GitHub Actions are not yet validated, Docker runtime verification is still pending, release notes need more review, or the maintainer wants a softer public signal before the first normal release.
