# v0.1.0 Release Notes Draft

Status: draft only. No tag, GitHub release, or published artifact has been created.

## Highlights

- SwiftUI + FastAPI starter kit for local-first iOS app development.
- Minimal backend auth flow with JWT access tokens.
- SwiftUI auth flow with Keychain token storage.
- Optional local-only Docker Compose/PostgreSQL setup with Alembic migrations.
- Codex-safe workflow docs, templates, review prompts, and handoff guidance.
- CI foundation for backend tests and iOS simulator build.
- Docs/script CI checks for shell syntax and local Markdown links.
- Expanded backend auth/security/config/session test coverage.
- Release-readiness, secret-audit, and example walkthrough documentation.

## Included

- FastAPI backend with:
  - `GET /health`
  - `POST /auth/signup`
  - `POST /auth/login`
  - `GET /users/me`
  - SQLAlchemy user model
  - Alembic initial migration
  - SQLite default tests
  - auth/security/config/session hardening tests
- SwiftUI iOS app with:
  - home screen
  - health check UI
  - signup/login forms
  - Keychain token storage
  - session restore
  - logout
  - lightweight local-only XCTest target
- Local Docker Compose with:
  - backend service
  - PostgreSQL service bound to localhost
  - placeholder-only environment values
- Documentation for:
  - architecture
  - backend and iOS design
  - local development
  - testing and CI
  - release readiness
  - secret audit
  - Codex-safe workflows

## Not Included

- Production deployment.
- Refresh tokens.
- Email verification.
- Password reset.
- OAuth.
- Roles or permissions.
- Production database configuration.
- Real App Store/TestFlight workflow.
- UI test automation.
- Production security audit.
- Enforced Ruff/SwiftFormat configuration.

## Safety Notes

- This starter is production-minded, not production-ready.
- Do not use placeholder credentials in production.
- Do not connect to production or external databases during starter verification.
- Do not use real user credentials in local examples.
- Do not run `docker compose down -v` unless you explicitly intend to delete the local PostgreSQL volume.
- GitHub releases, tags, deploys, pushes, and App Store/TestFlight operations require explicit human confirmation.

## Upgrade / Migration Notes

- This is the first planned public release, so there is no user-facing upgrade path yet.
- Local Docker/PostgreSQL users should run `docker compose exec backend alembic upgrade head`.
- Do not delete local Docker volumes unless you intentionally want to remove local test data.

## v0.1.0 vs v0.1.0-pre

Use `v0.1.0` if backend tests, iOS build, local-only iOS tests or `build-for-testing`, Docker/PostgreSQL verification, secret audit, generated artifact audit, and GitHub Actions are all reviewed.

Use `v0.1.0-pre` if GitHub Actions are not yet validated, Docker runtime verification is still pending, release notes need more review, or the maintainer wants a softer public signal before the first normal release.
