# Philosophy

`ai-app-starter-swiftui-fastapi` is designed for indie developers who want to build serious app products without starting from a blank folder.

The project favors clarity over cleverness. It should be understandable by one person, maintainable over time, and friendly to AI-assisted development without giving automation unrestricted control.

## Principles

- Production-minded, not production-heavy.
- Local-first before cloud-first.
- Documentation before automation.
- Small phases before large rewrites.
- Explicit safety rules before side effects.
- Clear contracts between iOS, backend, database, and auth.
- Useful defaults without hiding important decisions.

## What this starter kit should be

It should be:

- A practical starting point for a SwiftUI app backed by FastAPI.
- A teaching reference for common mobile backend architecture.
- A repo that Codex can work inside safely.
- A foundation that can evolve toward production.

## What this starter kit should not be

It should not be:

- A full SaaS platform.
- A one-command production deployment system.
- A replacement for security review.
- A place to store real secrets.
- A project where automation performs risky external actions without review.
